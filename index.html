Azure E-Commerce ETL Pipeline

A production-grade end-to-end ETL pipeline that ingests e-commerce data from a REST API, applies a Bronze → Silver → Gold medallion architecture on Azure Data Lake Storage Gen2, and serves analytics-ready tables via Azure Synapse Analytics using dbt. Orchestrated by Apache Airflow with parallel task execution, SLA monitoring, and Microsoft Teams alerting.

  • Medallion architecture (Bronze raw → Silver clean → Gold analytics-ready) with Hive-style partitioning for predicate pushdown
  • Idempotent re-runs via _row_hash deduplication; rejected rows written to audit path with rejection reason
  • Incremental dbt models (fct_orders, dim_customers) with CI/CD via GitHub Actions on every push

Streaming Pipeline Python · Azure Event Hubs · Synapse · Terraform · Prometheus

Azure Real-Time Stock Pipeline

A production-grade real-time streaming pipeline that ingests live stock price ticks via Azure Event Hubs, computes OHLCV micro-batch aggregations, applies technical indicators (SMA, RSI, Bollinger Bands), and serves enriched data from Azure Synapse Analytics. Infrastructure is fully reproducible via Terraform and monitored with Prometheus and Grafana.

  • Async stock tick producer publishing to Event Hubs with partition key per ticker for guaranteed ordering
  • Consumer computing OHLCV windows and writing via idempotent MERGE operations; offset-tracked via Blob Checkpoint Store
  • Stream processor computing SMA, RSI (Wilder's EWM), Bollinger Bands, and 3σ anomaly detection

Azure Data Quality Framework

A production-grade, reusable data quality framework built on Great Expectations, Apache Airflow, and Azure Synapse Analytics. Every table in the data platform has an expectation suite that runs daily, results are persisted for trending and SLA tracking, and failures trigger immediate Microsoft Teams alerts with full failure details.

  • Expectation suites as version-controlled Python (products, orders, stock OHLCV) running via Airflow DAG after ETL completion
  • DQ results written to Synapse for queryable trend history — success rate drops are visible over time, not just pass/fail per run
  • HTML report + Prometheus metrics endpoint (dq_success_pct, dq_failed_expectations) for Grafana dashboards
